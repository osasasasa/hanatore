# アイデア出しルール

## 基本方針

全アイデアを「どう稼ぐか」から逆算して設計する。
「インストール初日に課金が発生する構造」を必須条件とする。

## 3方向アプローチ

### 方向A：王道
- App Store / Google Play の売上上位カテゴリを分析
- 大手が支配していない隙間を特定
- 個人〜少人数チームで勝てるポジションを狙う
- 実績ある課金モデル（サブスク、フリーミアム）をベースに改善

### 方向B：逆張り
- 無料で提供されているが、本来お金を払う価値があるもの
- 課金導線が弱い人気アプリの「課金特化版」
- ニッチだが課金意欲が高いターゲット層

### 方向C：組み合わせ
- AI / パーソナライゼーション × サブスク
- 習慣形成 × 課金トリガー
- 既存カテゴリ × 新しいインタラクション

## アイデアの評価基準

各アイデアに以下を記載：

| 項目 | 内容 |
|------|------|
| アプリ名 | 英語名（グローバル展開可能、App Store検索で有利） |
| サブタイトル | App Storeサブタイトル案（30文字以内） |
| 一言説明 | 何をするアプリか |
| 課金モデル | サブスク / 買い切り / フリーミアム / 複合型 |
| 想定月額 or 価格 | 具体的な金額（日本円） |
| 想定LTV | 1ユーザーあたりの生涯価値 |
| 課金率の見込み | 無料→有料の転換率予測 |
| マネタイズの強さ | ◎/○/△ と理由 |
| 初日課金の仕組み | インストール当日に課金が発生する導線の概要 |
| 開発難易度 | 低/中/高 |
| 市場規模感 | 小/中/大 |

## アプリ名の命名基準

- 短く覚えやすい（2〜3音節が理想）
- App Store / Google Play で商標・既存アプリと被らなそうなもの
- ドメイン・SNSハンドルが取得可能そうなもの
- アプリの価値が名前から連想できる
- グローバル展開を想定し、英語ネイティブにも自然な響き

## 選定基準

「マネタイズの強さ × 開発難易度の低さ」で最もROIが高い1つを選ぶ。

選定時に考慮する追加要素：
- Day 0 課金の実現しやすさ
- リテンション（継続率）の見込み
- 競合の少なさ
- 技術的な差別化可能性
