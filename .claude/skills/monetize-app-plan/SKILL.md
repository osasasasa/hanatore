# マネタイズ重視アプリ企画スキル

マネタイズを最優先に設計したスマホアプリの企画・設計を、Claude Codeのplan形式で出力する。
「稼げるかどうか」を全ての意思決定の最上位基準とする。

## ワークフロー

### Phase 1: コンテキスト収集

まず最初に、ユーザーに以下を聞く：

**「今考えているアイデアや作りたいものはありますか？ざっくりでも構いません。」**

回答に応じて分岐：
- **アイデアがある場合** → そのアイデアを軸に深掘り。ステップ1のアイデア出しではユーザーのアイデアを最優先候補として含め、それを改善・発展させる案 + 別方向の比較案を出す
- **方向性だけある場合**（例：「ヘルスケア系」「AI使いたい」）→ その方向に絞ってアイデアを出す
- **何もない場合** → フルスクラッチで3方向からアイデアを出す

次に、以下を確認（未指定の場合はデフォルトを適用）：

| 項目 | デフォルト | 確認方法 |
|------|-----------|---------|
| 興味のあるジャンル | 指定なし（全方向で提案） | 質問 |
| 技術スタック | React Native (Expo) | 質問 |
| 課金基盤 | RevenueCat | 質問 |
| ターゲット市場 | 日本 + グローバル | 質問 |
| 開発期間の上限 | 6週間 | 質問 |
| 月間売上目標 | 未設定（シミュレーションで提示） | 質問 |
| AgentTeam使用予定 | Yes | 質問 |
| tasksコマンド使用予定 | Yes | 質問 |

### Phase 2: plan プロンプト生成

`references/plan-template.md` をベースに、Phase 1で収集した情報を埋め込んでplanプロンプトを生成する。

**生成ルール：**
- テンプレート内の `{{変数名}}` をPhase 1の回答で置換
- ジャンル指定がある場合、ステップ1のアイデア出し方向を調整
- AgentTeam不使用の場合、ステップ4-3とステップ5のAgent関連記述を削除
- tasksコマンド不使用の場合、タスク粒度の指定を削除

### Phase 3: plan 実行

生成したプロンプトをplanに渡し、以下の6ステップを順に実行：

1. **マネタイズファーストなアイデア出し** — 王道・逆張り・組み合わせの3方向で10個以上
2. **課金設計（最重要）** — 即日売上設計、RevenueCat実装、KPI、収益シミュレーション
3. **プロダクト定義** — 課金設計に従属するUX・MVP・ブランディング・競合分析
4. **技術設計** — アーキテクチャ、ディレクトリ構成、AgentTeam分割提案
5. **開発ロードマップ** — tasksコマンド変換前提のタスク分解
6. **CLAUDE.md生成** — プロジェクトルートに配置するファイル

各ステップの詳細は `references/plan-template.md` を参照。

### Phase 4: 出力と次ステップ案内

plan出力後、ユーザーに次のアクションを提示：

1. アイデアの選定確認（別のアイデアに変更可能）
2. AgentTeam構成の決定（提案から選択 or カスタマイズ）
3. tasksコマンドへの変換手順
4. 実装開始のコマンド例

## アイデア出しのルール

`references/idea-generation-rules.md` を参照。以下の3方向から合計10個以上：
- **王道**：市場実績 × 改善余地
- **逆張り**：取りこぼし課金機会
- **組み合わせ**：高LTV × 新技術

全アイデアに「インストール初日に課金が発生する構造」を必須条件とする。

おすすめを1つ選んで理由を説明する。

## 課金設計の原則

- 即日売上（Day 0 Revenue）を最優先で設計
- インストール→価値体験→課金を最短分数で到達させる
- RevenueCatのOffering/Entitlement/Experimentsをフル活用
- 収益シミュレーションは3パターン×12ヶ月で提示
